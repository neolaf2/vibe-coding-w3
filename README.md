## P3394 Agent Manifest Format

```json
{
  "agent_id": "your-agent-id",
  "version": "1.0.0",
  "capabilities": [
    {
      "name": "capability-name",
      "input_schema": {},
      "output_schema": {}
    }
  ],
  "channels": ["mcp", "rest"],
  "authorization": {
    "scopes": ["read", "write"]
  }
}
```

## Getting Started

### 1. Fork this repository
```bash
git checkout -b dev/<your-github-handle>
```

### 2. Prerequisites
Complete Weeks 1-2 before starting this week.

### 3. Key Prompts

**Module 3.1:** *"Choose a domain you care about. What are the 5 most important capabilities? Each becomes a skill. Draw the graph before you build."*

**Module 3.2:** *"Create a P3394 manifest for your domain agent. Declare capabilities, I/O schemas, and identity. Validate against the standard."*

**Module 3.3:** *"Build MCP + REST adapters. Send the same request through both. Any difference = abstraction leak."*

**Module 3.4:** *"Inject 3 failures (data, plan, auth). Use KSTAR traces to diagnose. Write a debugging narrative for the most interesting one."*

**Module 3.5 AAR:** *"Add audit logging to every action. Implement skill certification. Write an enterprise readiness assessment. AAR #3: assess capstone readiness."*
